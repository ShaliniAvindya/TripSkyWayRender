## ğŸ“ Key Concepts

### Temporary Password
```
What: Random 12-character password generated by system
Who: Sent to user via email
When: Only valid for 48 hours
Why: So admin never knows user's real password
Example: "X#aB7$mN2!pQ5"
Cannot be used as permanent password
Must be changed on first login
```

### Permanent Password
```
What: Password user creates and remembers
Who: Only known by the user
When: Set on first login (replacing temporary)
Why: User's own secure credential
Requirements: 12+ chars, UPPERCASE, lowercase, 123, !@#$
Expires: After 90 days
```

### Account Lifecycle
```
1. INVITED      (just created, invitation sent)
2. FIRST LOGIN  (user activates, sets permanent password)
3. ACTIVE       (ready to use)
4. RESET NEEDED (admin initiated password reset)
5. INACTIVE     (deactivated, cannot login)
```
